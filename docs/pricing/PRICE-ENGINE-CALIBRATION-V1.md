## 1. What F1.5 is

F1.5 is the small bridge between F1 (architecture) and F2 (schema migration). It produces three artifacts:

1. **Five empty CSV templates** (v2; was two in v1) — `format_defaults_v1_eur.csv` (63 rows) + `platform_floors_v1_eur.csv` (63 rows) + `sublabel_multipliers_v1.csv` (17 rows) + `use_tier_multipliers_v1.csv` (12 rows) + `cc_variants_v1_eur.csv` (7 rows) — all generated by the bootstrap script
2. **The calibration process** — what sources to consult, how to fill, what to validate, which cells need extra care
3. **A converter script** — reads the founder-filled CSVs and emits a SQL seed migration (5 INSERT blocks; one per table) that F2 includes verbatim

After founder fills the CSVs and the converter produces the SQL seed:
- F2 schema migration directive composes against a fully-resolved table content
- Founder reviews the SQL seed migration before F2 merges
- F3 (`format_defaults` adapter implementation) ships against ratified data

F1.5 ships dormant — no engine code; no migration; no UI. It's a tooling + spec PR. The actual calibration is founder offline work.

## 6. CSV template — `format_defaults_v1_eur.csv` (v2)

### 6.1 Schema

One row per (format, intrusion_level, licence_class) combination at currency=EUR. Total: 7 × 3 × 3 = **63 rows** (was 252 in v1; reduced by moving use_tier to a separate multiplier table per L1 v2 §5.3).

| Column | Type | Notes |
|---|---|---|
| `format` | string | One of: `photo`, `illustration`, `infographic`, `vector`, `video`, `audio`, `text` |
| `intrusion_level` | string | One of: `light`, `standard`, `heavy` |
| `licence_class` | string | **v2:** one of `editorial`, `commercial`, `advertising` (was `Social`/`Editorial`/`Campaign` in v1; CC excluded — handled via `cc_variants_v1_eur.csv`) |
| `currency` | string | `EUR` (constant in v1) |
| `baseline_cents` | integer | Founder-filled; positive integer in cents (e.g., 25000 = €250.00). This is the BASE price for the cell at the *anchor* use_tier (tier_2 by convention; multiplier 1.000); other tiers derive via `pricing_use_tier_multipliers`. |
| `calibration_basis` | string | Short founder-written note (e.g., "fotoQuote 2024 editorial mid-pub midpoint × 0.85 FF positioning"); 1-200 chars |

### 6.2 Header row + first row example (illustrative; founder fills actual values)

```csv
format,intrusion_level,licence_class,currency,baseline_cents,calibration_basis
photo,light,editorial,EUR,,
photo,light,commercial,EUR,,
photo,light,advertising,EUR,,
photo,standard,editorial,EUR,,
... (59 more rows — bootstrap script generates all 63)
```

Founder fills `baseline_cents` + `calibration_basis` per row. The bootstrap script pre-fills `currency` since it's constant.

### 6.3 Per-cell guidance (founder offline reference)

The 63 cells aren't independent — there are intuitive ratios across dimensions. Founder may calibrate by anchoring a few "spine" cells and deriving others by ratio. The directive does NOT prescribe ratios — that's editorial judgment — but offers the spine-cell anchoring approach as a possible workflow:

| Spine cell candidate (v2) | Why anchor here |
|---|---|
| `photo / standard / editorial` | The most common asset shape on Frontfiles; market data is densest here; anchor first |
| `photo / standard / commercial` | Defines the editorial → commercial spread |
| `photo / standard / advertising` | Defines the commercial → advertising spread (advertising commands premium) |
| `photo / heavy / editorial` | Defines the intrusion-level spread within editorial |

Once 3-5 spine cells are anchored, the remaining 58 cells can be derived by ratio + sanity-check. F1 §3.3 calibration sources guide the spine values. **CC pricing is calibrated separately in `cc_variants_v1_eur.csv` per §6A** (absolute prices per variant).

### 6.4 Validation rules (enforced by the converter; v2)

- All 63 rows present (no missing combinations)
- `format`, `intrusion_level`, `licence_class` values match the enumerated set (no typos, no bonus values)
- `licence_class` ∈ `{editorial, commercial, advertising}` only — `creative_commons` REJECTED here (belongs in `cc_variants_v1_eur.csv`)
- `currency` = `EUR` on every row
- `baseline_cents` is a positive integer (no nulls, no zeros, no decimals)
- `baseline_cents` is within sanity bounds: 100 ≤ value ≤ 10000000 (€1.00 to €100,000.00); flagged if outside, hard-error if < 0
- `calibration_basis` is non-empty (≥ 5 chars); helps recalibration teams understand the row's source
- No duplicate `(format, intrusion_level, licence_class)` combinations

The converter prints validation warnings + errors before generating SQL. Errors block; warnings surface for founder review.

### 6A.3 Per-cell guidance

CC0 = 0 is canonical. The other 6 variants encode increasing rights restriction:
- CC-BY: most permissive (attribution only); typical small admin fee
- CC-BY-SA: attribution + share-alike; similar to CC-BY in price
- CC-BY-ND: attribution + no derivatives; modest premium (no remixing)
- CC-BY-NC: attribution + non-commercial; commercial-use buyers pay; non-commercial use is free under CC
- CC-BY-NC-SA / CC-BY-NC-ND: most restrictive; commercial-use buyers pay highest CC tier

### 7.3 Per-cell guidance (v2)

Floors should sit **strictly below** the corresponding `format_defaults` cell at the lowest use_tier multiplier. Specifically: for any `(format, intrusion_level, licence_class)` cell in `platform_floors`, the `min_cents` should be ≤ `baseline_cents` × `min(use_tier_multiplier across 4 tiers)` for the matching `(format, intrusion_level, licence_class)` cell in `format_defaults`. The converter's cross-CSV check (§7.4) enforces this as a warning.

If floors are set higher than format_defaults, the engine clamps every recommendation up — meaning format_defaults effectively never applies. That's almost always wrong.

The converter flags floor > min(defaults) as a warning (not an error) so founder can decide.

### 7.4 Validation rules

Same shape as §6.4:
- All 63 rows present
- Enumerated values match
- `currency` = `EUR`
- `min_cents` positive integer; sanity-bounded 100 ≤ x ≤ 10000000
- Cross-CSV check: `min_cents` ≤ `min(baseline_cents across use_tiers)` for matching `(format, intrusion_level, licence_class)` — warning, not error

## 8. Calibration sources

Per F1 §3.3 and brief v3 §4.2.4, **external sources inform initial calibration but are never cited as authoritative** in the basis statements creators see. The `calibration_basis` column captures the founder's reasoning for audit + future recalibration; the engine's runtime basis statement uses Frontfiles-positioned language ("Frontfiles standard rate for editorial photo at standard intrusion").

**Recommended sources to consult during the calibration pass:**

| Source | Use | Notes |
|---|---|---|
| **fotoQuote / fotoBiz** | Editorial + commercial photography pricing midpoints | Most directly applicable to photo + illustration cells; explicit per-use-tier breakdowns |
| **Getty Images published rate cards** | Editorial / news rates where public | Use as upper-bound reference for major_pub cells |
| **AP / Reuters published editorial rates** | Editorial wire rates | Anchor the small_pub / mid_pub editorial cells |
| **PhotoShelter pricing guides** | Cross-format photographer-side guidance | Useful for illustration + vector format cells |
| **Adobe Stock / Shutterstock published prices** | Commercial bulk-licence reference | Use as lower-bound reference; FF's commercial cells should sit above stock-pricing |
| **Founder's market knowledge** | FF positioning relative to incumbents | The actual editorial choice; sources inform but founder decides |

For non-photo formats (video, audio, text):
- Video: published broadcast / streaming licence rates
- Audio: PRS / ASCAP / BMI rate schedules where public; SoundCloud / pondfive references
- Text: AP / Reuters article rates; Society of Authors editorial rates

If a cell doesn't have a clean external reference (e.g., `text / heavy / advertising`), founder uses judgment + ratio derivation from spine cells.

## 9. Bootstrap script — `scripts/pricing/bootstrap-calibration-csvs.ts`

### 9.1 Behavior

```typescript
// Pseudocode (v2 — generates 5 CSVs: format_defaults, platform_floors, sublabel_multipliers, use_tier_multipliers, cc_variants)

import {
  LICENCE_SUBLABELS,  // 24 dotted values from src/lib/licence/types.ts (L2 §7.1)
  CC_VARIANTS,        // 7 values
  USE_TIERS,          // ['tier_1', 'tier_2', 'tier_3', 'tier_4']
} from '@/lib/licence/types'

const FORMATS = ['photo', 'illustration', 'infographic', 'vector', 'video', 'audio', 'text']
const INTRUSION_LEVELS = ['light', 'standard', 'heavy']
const CELL_LICENCE_CLASSES = ['editorial', 'commercial', 'advertising'] // v2: CC excluded — has its own table

// 1. Generate format_defaults CSV — 63 rows (was 252 in v1; use_tier dropped)
const defaults: string[] = ['format,intrusion_level,licence_class,currency,baseline_cents,calibration_basis']
for (const f of FORMATS)
  for (const i of INTRUSION_LEVELS)
    for (const l of CELL_LICENCE_CLASSES)
      defaults.push(`${f},${i},${l},EUR,,`)

// 2. Generate platform_floors CSV — 63 rows (unchanged in v2)
// Note: no calibration_basis column for floors per §7.1 (F1 §4.1 schema asymmetry)
const floors: string[] = ['format,intrusion_level,licence_class,currency,min_cents']
for (const f of FORMATS)
  for (const i of INTRUSION_LEVELS)
    for (const l of CELL_LICENCE_CLASSES)
      floors.push(`${f},${i},${l},EUR,`)

// 3. Generate sublabel_multipliers CSV — 17 rows (NEW v2; non-CC sublabels only)
const sublabelMultipliers: string[] = ['sublabel,multiplier']
for (const sl of LICENCE_SUBLABELS) {
  if (sl.startsWith('creative_commons.')) continue  // CC handled separately
  // Pre-fill 'editorial.news' as anchor (1.000); leave rest blank
  const preset = sl === 'editorial.news' ? '1.000' : ''
  sublabelMultipliers.push(`${sl},${preset}`)
}

// 4. Generate use_tier_multipliers CSV — 12 rows (NEW v2; CC excluded)
const useTierMultipliers: string[] = ['licence_class,use_tier,multiplier']
for (const c of CELL_LICENCE_CLASSES)
  for (const t of USE_TIERS) {
    // Pre-fill tier_2 = 1.000 per class as the mid anchor
    const preset = t === 'tier_2' ? '1.000' : ''
    useTierMultipliers.push(`${c},${t},${preset}`)
  }

// 5. Generate cc_variants CSV — 7 rows (NEW v2; flat per variant in v1)
const ccVariants: string[] = ['cc_variant,currency,price_cents']
for (const v of CC_VARIANTS) {
  // Pre-fill cc0 = 0 (canonical public-domain dedication); leave rest blank
  const preset = v === 'cc0' ? '0' : ''
  ccVariants.push(`${v},EUR,${preset}`)
}

// Write all 5 atomically; per-file refuse-to-overwrite-when-filled semantics:
//   - if file exists AND contains any user-filled values, abort with clear message
//   - if file exists AND only contains pre-filled anchors (editorial.news=1.000, tier_2=1.000, cc0=0), overwrite
//   - if file does not exist, write
```

### 9.3 Why a bootstrap script (not just commit empty CSVs directly)

The dimension enumerations might drift between F1.5 composition and F2 ship. Bootstrapping from the actual `AssetFormat` enum + the documented dimension lists keeps the CSVs aligned with code. If `AssetFormat` adds a new value (e.g., `3d`), re-running bootstrap regenerates the CSV with the additional rows for the new format — founder fills only the new cells.

The bootstrap reads the canonical enumeration from `src/lib/types.ts` if available; falls back to the literal lists in §9.1 if the file doesn't have an exported enum yet. Either way the source-of-truth-divergence risk is minimized.

## 10. Converter script — `scripts/pricing/csv-to-seed-migration.ts`

### 10.1 Behavior (v2)

Reads the **five** filled CSVs and emits a SQL seed migration with **5 INSERT blocks**. Validation runs first; errors block; warnings print but allow continuation.

```typescript
// Pseudocode (v2)

async function main() {
  const defaultsCsv = readCsv('docs/pricing/calibration/format_defaults_v1_eur.csv')           // 63 rows
  const floorsCsv = readCsv('docs/pricing/calibration/platform_floors_v1_eur.csv')             // 63 rows
  const sublabelMultCsv = readCsv('docs/pricing/calibration/sublabel_multipliers_v1.csv')      // 17 rows
  const useTierMultCsv = readCsv('docs/pricing/calibration/use_tier_multipliers_v1.csv')       // 12 rows
  const ccVariantsCsv = readCsv('docs/pricing/calibration/cc_variants_v1_eur.csv')             // 7 rows

  // Validation pass — per-CSV
  const errors = []
  const warnings = []
  validateRows(defaultsCsv, /* §6.4 */, errors, warnings)
  validateRows(floorsCsv, /* §7.4 */, errors, warnings)
  validateRows(sublabelMultCsv, /* §6B.4 */, errors, warnings)
  validateRows(useTierMultCsv, /* §6C.4 */, errors, warnings)
  validateRows(ccVariantsCsv, /* §6A.4 */, errors, warnings)
  validateFloorsBelowDefaults(floorsCsv, defaultsCsv, useTierMultCsv, warnings)  // §7.3 cross-check (uses min use_tier multiplier)
  validateCc0IsZero(ccVariantsCsv, warnings)  // §6A.4 — CC0 should be 0
  if (errors.length > 0) {
    console.error('VALIDATION ERRORS — converter aborted:')
    errors.forEach(e => console.error('  ' + e))
    process.exit(1)
  }
  if (warnings.length > 0) {
    console.warn('Warnings (proceeding anyway):')
    warnings.forEach(w => console.warn('  ' + w))
  }

  const tableVersion = 1
  const calibratedAt = new Date().toISOString()
  const calibratedBy = process.env.CALIBRATED_BY_USER_ID

  if (!calibratedBy) {
    console.error('CALIBRATED_BY_USER_ID env var required')
    process.exit(1)
  }

  let sql = `-- F2 seed (v2): 5 pricing tables (EUR)\n`
  sql += `-- Generated by scripts/pricing/csv-to-seed-migration.ts on ${calibratedAt}\n\n`

  // Block 1 — pricing_format_defaults (63 rows; v2: no use_tier column)
  sql += `INSERT INTO pricing_format_defaults\n`
  sql += `  (format, intrusion_level, licence_class, currency, baseline_cents, table_version, effective_from, calibration_basis, calibrated_at, calibrated_by)\n`
  sql += `VALUES\n`
  sql += defaultsCsv.map(r =>
    `  ('${r.format}', '${r.intrusion_level}', '${r.licence_class}', '${r.currency}', ${r.baseline_cents}, ${tableVersion}, now(), ${quoteString(r.calibration_basis)}, now(), '${calibratedBy}')`
  ).join(',\n') + ';\n\n'

  // Block 2 — pricing_platform_floors (63 rows)
  sql += `INSERT INTO pricing_platform_floors\n`
  sql += `  (format, intrusion_level, licence_class, currency, min_cents, effective_from, calibrated_at, calibrated_by)\n`
  sql += `VALUES\n`
  sql += floorsCsv.map(r =>
    `  ('${r.format}', '${r.intrusion_level}', '${r.licence_class}', '${r.currency}', ${r.min_cents}, now(), now(), '${calibratedBy}')`
  ).join(',\n') + ';\n\n'

  // Block 3 — pricing_sublabel_multipliers (17 rows; per L2 §6.3)
  sql += `INSERT INTO pricing_sublabel_multipliers\n`
  sql += `  (sublabel, multiplier, effective_from, calibrated_at, calibrated_by)\n`
  sql += `VALUES\n`
  sql += sublabelMultCsv.map(r =>
    `  ('${r.sublabel}', ${r.multiplier}, now(), now(), '${calibratedBy}')`
  ).join(',\n') + ';\n\n'

  // Block 4 — pricing_use_tier_multipliers (12 rows; per L2 §6.4)
  sql += `INSERT INTO pricing_use_tier_multipliers\n`
  sql += `  (licence_class, use_tier, multiplier, effective_from, calibrated_at, calibrated_by)\n`
  sql += `VALUES\n`
  sql += useTierMultCsv.map(r =>
    `  ('${r.licence_class}', '${r.use_tier}', ${r.multiplier}, now(), now(), '${calibratedBy}')`
  ).join(',\n') + ';\n\n'

  // Block 5 — pricing_cc_variants (7 rows; per L2 §6.2)
  sql += `INSERT INTO pricing_cc_variants\n`
  sql += `  (cc_variant, currency, price_cents, effective_from, calibrated_at, calibrated_by)\n`
  sql += `VALUES\n`
  sql += ccVariantsCsv.map(r =>
    `  ('${r.cc_variant}', '${r.currency}', ${r.price_cents}, now(), now(), '${calibratedBy}')`
  ).join(',\n') + ';\n'

  writeFile('supabase/migrations/_pending/<TIMESTAMP>_pricing_seed_v1.sql', sql)
  console.log('Wrote SQL seed (5 blocks; 162 total rows) to _pending/.')
}
```

### 10.2 Output location

Generated SQL lands in `supabase/migrations/_pending/` (a directory the converter creates if absent). F2 directive will move it to `supabase/migrations/` with a timestamp prefix at F2 ship time.

The `_pending/` directory is gitignored or kept out of the migration runner's path so accidental untimed-migration application doesn't happen.

### 10.3 Re-run semantics

Re-running the converter with edited CSVs regenerates `_pending/` SQL with `table_version = 1`. **The version bump is manual** — when calibration is recalibrated post-launch, the founder bumps the version constant in the converter script (or passes `--version=2` flag) and the new SQL preserves old rows + adds new rows per F1 §3.3 recalibration cadence.

For v1 seed: `table_version = 1` is the only value the converter produces.

### 10.4 SQL escaping

The `calibration_basis` strings may contain quotes, commas, etc. The converter uses parameterized SQL OR proper SQL escaping (`E'...'` strings with backslash escapes; or `$$...$$` dollar-quoting). The reference implementation must handle:
- Single quotes in basis text (e.g., "fotoQuote's 2024 editorial midpoint")
- Newlines in basis text (rare; escape as `\n`)
- Unicode in basis text (UTF-8 input → UTF-8 output)

## 11. Founder workflow (the actual calibration pass)

```
Phase 1 — Preparation (~30 min)
  1. Read this directive (F1.5) end to end
  2. Read F1 §3 + §4 (table definitions) + brief v3 §4.2.3 (format_defaults adapter)
  3. Open the CSV templates in your editor / spreadsheet of choice
  4. Skim the calibration sources (§8) and pick which 2-3 you'll consult

Phase 2 — Spine cells in format_defaults (~1 hr)
  5. Anchor 3-5 spine cells per §6.3 (v2):
     - photo / standard / editorial   (most common; market data densest)
     - photo / standard / commercial  (editorial → commercial spread)
     - photo / standard / advertising (commercial → advertising premium)
     - photo / heavy / editorial      (intrusion-level spread)
  6. For each spine cell: consult sources; pick value; write calibration_basis

Phase 3 — Derive remaining format_defaults (~2-3 hr)
  7. Fill remaining photo cells by ratio from the spine
  8. Fill illustration cells (often photo × 0.7-1.2 depending on creator effort)
  9. Fill infographic cells (typically higher than photo for editorial)
  10. Fill vector cells (usually below illustration; format simplicity)
  11. Fill video / audio / text cells using format-specific external sources

Phase 4 — Floors (~20 min)
  12. Fill platform_floors_v1_eur.csv per §7.3 (≤ baseline_cents × min(use_tier_multiplier))
  13. Cross-check: every floor cell ≤ baseline × min(use_tier_multiplier) for matching (format, intrusion, licence_class)

Phase 5 — Sublabel multipliers (~1 hr)
  14. Fill sublabel_multipliers_v1.csv per §6B.3 (17 multipliers)
  15. Anchor: editorial.news = 1.000 (already pre-filled by bootstrap)
  16. Derive other sublabels by ratio (e.g., documentary = 1.4 × news)

Phase 6 — Use-tier multipliers (~30 min)
  17. Fill use_tier_multipliers_v1.csv per §6C.3 (12 multipliers)
  18. Anchor: tier_2 = 1.000 per class (pre-filled by bootstrap)
  19. Derive tier_1 (smaller) and tier_3, tier_4 (larger) per class

Phase 7 — CC variant absolute prices (~30 min)
  20. Fill cc_variants_v1_eur.csv per §6A.3 (7 variants)
  21. CC0 = 0 (canonical; pre-filled). Decide CC-BY admin fee, CC-BY-NC commercial-use price, etc.

Phase 8 — Validation + seed (~10 min Claude work after CSVs filled)
  22. Run scripts/pricing/csv-to-seed-migration.ts; resolve any validation errors
  23. Review the generated SQL seed (5 INSERT blocks)
  24. Sanity-check 5-10 random cells against the source

Phase 9 — Ratify F1.5 v2 + ship F2 (Claude composes F2 directive after this gate)
  25. Founder reviews the filled CSVs as a docs PR
  26. Approve the calibration; F2 schema migration directive composes against now-known seed values
```

## 14. Don't-do list

1. **Don't fill CSVs in F1.5's PR.** This directive ships the templates + tooling; calibration values are Stage B (founder offline work + separate PR).
2. **Don't skip the validation pass.** The converter's validation catches typos, missing rows, out-of-bound values — these are the most common errors when filling 5 CSVs by hand (162 total rows in v2).
3. **Don't add multi-currency support here.** EUR-only v1 per F1 §5.2. Multi-currency is per-currency calibration directives (F1.5-USD, F1.5-GBP, etc.).
4. **Don't cite external sources as authoritative in `calibration_basis`.** The column is internal documentation for recalibration teams. Per brief v3 §4.2.4, the engine's runtime basis statement to creators uses Frontfiles-positioned language; the calibration_basis is the founder's note for future-self / future-team.
5. **Don't auto-generate "spine" cell values.** Founder picks the anchors. The directive offers spine-cell candidates as workflow guidance, not as required values.
6. **Don't ship the generated SQL outside `_pending/` until F2.** F2 directive timestamps + moves the seed into `supabase/migrations/`.
7. **Don't bypass the floors cross-check.** Even though it's a warning (not an error), every flagged cell deserves explicit founder review before F2.
8. **Don't add per-format exclusivity rows.** Exclusivity is downstream of base price per `EXCLUSIVE_MULTIPLIERS` + brief v3 §4.6 + F1 §3.1. F1.5's CSVs do NOT have an `exclusivity` column.
9. **Don't introduce floors below the platform's commercial reality.** A `min_cents` of €1.00 effectively disables the floor for that combination. Floors should be set at meaningful values (typical: 30-70% of the corresponding small_pub default, depending on format).
10. **Don't merge F2 before this directive ratifies AND the calibration values are filled.** Both gates required per §13.
