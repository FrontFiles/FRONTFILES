## 1. What F1.5 is

F1.5 is the small bridge between F1 (architecture) and F2 (schema migration). It produces three artifacts:

1. **Two empty CSV templates** — `format_defaults_v1_eur.csv` (252 rows) + `platform_floors_v1_eur.csv` (63 rows) — generated by a small bootstrap script Claude writes
2. **The calibration process** — what sources to consult, how to fill, what to validate, which cells need extra care
3. **A converter script** — reads the founder-filled CSVs and emits a SQL seed migration that F2 includes verbatim

After founder fills the CSVs and the converter produces the SQL seed:
- F2 schema migration directive composes against a fully-resolved table content
- Founder reviews the SQL seed migration before F2 merges
- F3 (`format_defaults` adapter implementation) ships against ratified data

F1.5 ships dormant — no engine code; no migration; no UI. It's a tooling + spec PR. The actual calibration is founder offline work.

## 6. CSV template — `format_defaults_v1_eur.csv`

### 6.1 Schema

One row per (format, intrusion_level, licence_class, use_tier) combination at currency=EUR. Total: 7 × 3 × 3 × 4 = 252 rows.

| Column | Type | Notes |
|---|---|---|
| `format` | string | One of: `photo`, `illustration`, `infographic`, `vector`, `video`, `audio`, `text` |
| `intrusion_level` | string | One of: `light`, `standard`, `heavy` |
| `licence_class` | string | One of: `Social`, `Editorial`, `Campaign` |
| `use_tier` | string | One of: `small_pub`, `mid_pub`, `major_pub`, `commercial` |
| `currency` | string | `EUR` (constant in v1) |
| `baseline_cents` | integer | Founder-filled; positive integer in cents (e.g., 25000 = €250.00) |
| `calibration_basis` | string | Short founder-written note (e.g., "fotoQuote 2024 editorial small-pub midpoint × 0.85 FF positioning"); 1-200 chars |

### 6.2 Header row + first row example (illustrative; founder fills actual values)

```csv
format,intrusion_level,licence_class,use_tier,currency,baseline_cents,calibration_basis
photo,light,Social,small_pub,EUR,,
photo,light,Social,mid_pub,EUR,,
photo,light,Social,major_pub,EUR,,
photo,light,Social,commercial,EUR,,
photo,light,Editorial,small_pub,EUR,,
... (247 more rows — bootstrap script generates all 252)
```

Founder fills `baseline_cents` + `calibration_basis` per row. The bootstrap script can pre-fill `currency` since it's constant.

### 6.3 Per-cell guidance (founder offline reference)

The 252 cells aren't independent — there are intuitive ratios across dimensions. Founder may calibrate by anchoring a few "spine" cells and deriving others by ratio. The directive does NOT prescribe ratios — that's editorial judgment — but offers the spine-cell anchoring approach as a possible workflow:

| Spine cell candidate | Why anchor here |
|---|---|
| `photo / standard / Editorial / mid_pub` | The most common cell in editorial usage; market data is densest here; anchor first |
| `photo / standard / Editorial / major_pub` | Defines the publication-size scale |
| `photo / standard / Campaign / commercial` | Defines the licence-class spread |
| `photo / heavy / Editorial / mid_pub` | Defines the intrusion-level spread |

Once 3-5 spine cells are anchored, the remaining 247 cells can be derived by ratio + sanity-check. F1 §3.3 calibration sources guide the spine values.

### 6.4 Validation rules (enforced by the converter)

- All 252 rows present (no missing combinations)
- `format`, `intrusion_level`, `licence_class`, `use_tier` values match the enumerated set (no typos, no bonus values)
- `currency` = `EUR` on every row
- `baseline_cents` is a positive integer (no nulls, no zeros, no decimals)
- `baseline_cents` is within sanity bounds: 100 ≤ value ≤ 10000000 (€1.00 to €100,000.00); flagged if outside, hard-error if < 0
- `calibration_basis` is non-empty (≥ 5 chars); helps recalibration teams understand what the row's source was
- No duplicate `(format, intrusion_level, licence_class, use_tier)` combinations

The converter prints validation warnings + errors before generating SQL. Errors block; warnings surface for founder review.

## 7. CSV template — `platform_floors_v1_eur.csv`

### 7.1 Schema

One row per (format, intrusion_level, licence_class) combination at currency=EUR. Total: 7 × 3 × 3 = 63 rows. **No `use_tier`** dimension — floors are below-which-the-engine-never-recommends regardless of use tier.

| Column | Type | Notes |
|---|---|---|
| `format` | string | Same enumeration as §6.1 |
| `intrusion_level` | string | Same |
| `licence_class` | string | Same |
| `currency` | string | `EUR` |
| `min_cents` | integer | Founder-filled; positive integer |

**Note on `calibration_basis` for floors:** F1 §4.1's `pricing_platform_floors` schema does NOT include a `calibration_basis` column (unlike `pricing_format_defaults` which does). This is an asymmetry in F1 — floors are simpler than defaults so the per-row audit text was omitted. F1.5 follows F1's schema verbatim; the floors CSV does not include `calibration_basis`. If the audit utility is wanted for floors, a small F1 corrigendum can add the column in a follow-on; not blocking F1.5 ratification or F2 composition.

### 7.2 Header row + first row example

```csv
format,intrusion_level,licence_class,currency,min_cents
photo,light,Social,EUR,
photo,light,Editorial,EUR,
photo,light,Campaign,EUR,
photo,standard,Social,EUR,
... (59 more rows — bootstrap script generates all 63)
```

### 7.3 Per-cell guidance

Floors should sit **strictly below** the corresponding `format_defaults` cells (any use_tier). Specifically: for any `(format, intrusion_level, licence_class)` cell in `platform_floors`, the `min_cents` should be ≤ the minimum `baseline_cents` across the 4 `use_tier` rows in `format_defaults` for that (format, intrusion_level, licence_class) combination.

If floors are set higher than format_defaults, the engine clamps every recommendation up — meaning format_defaults effectively never applies. That's almost always wrong.

The converter flags floor > min(defaults) as a warning (not an error) so founder can decide.

### 7.4 Validation rules

Same shape as §6.4:
- All 63 rows present
- Enumerated values match
- `currency` = `EUR`
- `min_cents` positive integer; sanity-bounded 100 ≤ x ≤ 10000000
- Cross-CSV check: `min_cents` ≤ `min(baseline_cents across use_tiers)` for matching `(format, intrusion_level, licence_class)` — warning, not error

## 8. Calibration sources

Per F1 §3.3 and brief v3 §4.2.4, **external sources inform initial calibration but are never cited as authoritative** in the basis statements creators see. The `calibration_basis` column captures the founder's reasoning for audit + future recalibration; the engine's runtime basis statement uses Frontfiles-positioned language ("Frontfiles standard rate for editorial photo at standard intrusion").

**Recommended sources to consult during the calibration pass:**

| Source | Use | Notes |
|---|---|---|
| **fotoQuote / fotoBiz** | Editorial + commercial photography pricing midpoints | Most directly applicable to photo + illustration cells; explicit per-use-tier breakdowns |
| **Getty Images published rate cards** | Editorial / news rates where public | Use as upper-bound reference for major_pub cells |
| **AP / Reuters published editorial rates** | Editorial wire rates | Anchor the small_pub / mid_pub editorial cells |
| **PhotoShelter pricing guides** | Cross-format photographer-side guidance | Useful for illustration + vector format cells |
| **Adobe Stock / Shutterstock published prices** | Commercial bulk-licence reference | Use as lower-bound reference; FF's commercial cells should sit above stock-pricing |
| **Founder's market knowledge** | FF positioning relative to incumbents | The actual editorial choice; sources inform but founder decides |

For non-photo formats (video, audio, text):
- Video: published broadcast / streaming licence rates
- Audio: PRS / ASCAP / BMI rate schedules where public; SoundCloud / pondfive references
- Text: AP / Reuters article rates; Society of Authors editorial rates

If a cell doesn't have a clean external reference (e.g., `text / heavy / Campaign / commercial`), founder uses judgment + ratio derivation from spine cells.

## 9. Bootstrap script — `scripts/pricing/bootstrap-calibration-csvs.ts`

### 9.1 Behavior

```typescript
// Pseudocode

const FORMATS = ['photo', 'illustration', 'infographic', 'vector', 'video', 'audio', 'text']
const INTRUSION_LEVELS = ['light', 'standard', 'heavy']
const LICENCE_CLASSES = ['Social', 'Editorial', 'Campaign']
const USE_TIERS = ['small_pub', 'mid_pub', 'major_pub', 'commercial']

// Generate format_defaults CSV — 252 rows
const defaults: string[] = ['format,intrusion_level,licence_class,use_tier,currency,baseline_cents,calibration_basis']
for (const f of FORMATS)
  for (const i of INTRUSION_LEVELS)
    for (const l of LICENCE_CLASSES)
      for (const u of USE_TIERS)
        defaults.push(`${f},${i},${l},${u},EUR,,`)

// Generate platform_floors CSV — 63 rows
// Note: no calibration_basis column for floors per §7.1 (F1 §4.1 schema asymmetry)
const floors: string[] = ['format,intrusion_level,licence_class,currency,min_cents']
for (const f of FORMATS)
  for (const i of INTRUSION_LEVELS)
    for (const l of LICENCE_CLASSES)
      floors.push(`${f},${i},${l},EUR,`)

// Write atomically; refuse to overwrite if already filled
//   - if file exists AND contains any non-empty baseline_cents/min_cents, abort with clear message
//   - if file exists AND all rows are still blank, overwrite (idempotent re-bootstrap)
//   - if file does not exist, write
```

### 9.2 Verification

```bash
bun run scripts/pricing/bootstrap-calibration-csvs.ts
# Expected: writes both CSVs to docs/pricing/calibration/; prints "wrote 252 default rows; wrote 63 floor rows"

# Re-run with empty CSVs:
bun run scripts/pricing/bootstrap-calibration-csvs.ts
# Expected: same output (idempotent)

# Re-run with partially-filled CSVs:
bun run scripts/pricing/bootstrap-calibration-csvs.ts
# Expected: aborts with "X rows have values; refuse to overwrite. Delete the CSV manually if you want to start over."
```

### 9.3 Why a bootstrap script (not just commit empty CSVs directly)

The dimension enumerations might drift between F1.5 composition and F2 ship. Bootstrapping from the actual `AssetFormat` enum + the documented dimension lists keeps the CSVs aligned with code. If `AssetFormat` adds a new value (e.g., `3d`), re-running bootstrap regenerates the CSV with the additional rows for the new format — founder fills only the new cells.

The bootstrap reads the canonical enumeration from `src/lib/types.ts` if available; falls back to the literal lists in §9.1 if the file doesn't have an exported enum yet. Either way the source-of-truth-divergence risk is minimized.

## 10. Converter script — `scripts/pricing/csv-to-seed-migration.ts`

### 10.1 Behavior

Reads the two filled CSVs and emits a SQL seed migration. Validation runs first; errors block; warnings print but allow continuation.

```typescript
// Pseudocode

async function main() {
  const defaultsCsv = readCsv('docs/pricing/calibration/format_defaults_v1_eur.csv')
  const floorsCsv = readCsv('docs/pricing/calibration/platform_floors_v1_eur.csv')

  // Validation pass
  const errors = []
  const warnings = []
  validateRows(defaultsCsv, /* row schema per §6.4 */, errors, warnings)
  validateRows(floorsCsv, /* row schema per §7.4 */, errors, warnings)
  validateFloorsBelowDefaults(floorsCsv, defaultsCsv, warnings) // §7.3 cross-check
  if (errors.length > 0) {
    console.error('VALIDATION ERRORS — converter aborted:')
    errors.forEach(e => console.error('  ' + e))
    process.exit(1)
  }
  if (warnings.length > 0) {
    console.warn('Warnings (proceeding anyway):')
    warnings.forEach(w => console.warn('  ' + w))
  }

  // Generate SQL seed
  const tableVersion = 1  // first calibration; bumps on each recalibration cycle
  const calibratedAt = new Date().toISOString()
  const calibratedBy = process.env.CALIBRATED_BY_USER_ID  // founder UUID; required env

  if (!calibratedBy) {
    console.error('CALIBRATED_BY_USER_ID env var required (the founder/admin user UUID)')
    process.exit(1)
  }

  let sql = `-- F2 seed: pricing_format_defaults v1 (EUR; ${defaultsCsv.length} rows)\n`
  sql += `-- Generated by scripts/pricing/csv-to-seed-migration.ts on ${calibratedAt}\n`
  sql += `INSERT INTO pricing_format_defaults\n`
  sql += `  (format, intrusion_level, licence_class, use_tier, currency, baseline_cents, table_version, effective_from, calibration_basis, calibrated_at, calibrated_by)\n`
  sql += `VALUES\n`
  sql += defaultsCsv.map(r =>
    `  ('${r.format}', '${r.intrusion_level}', '${r.licence_class}', '${r.use_tier}', '${r.currency}', ${r.baseline_cents}, ${tableVersion}, now(), ${quoteString(r.calibration_basis)}, now(), '${calibratedBy}')`
  ).join(',\n') + ';\n\n'

  // Same for platform_floors (note: no calibration_basis column; see §7.1)
  sql += `-- F2 seed: pricing_platform_floors v1 (EUR; ${floorsCsv.length} rows)\n`
  sql += `INSERT INTO pricing_platform_floors\n`
  sql += `  (format, intrusion_level, licence_class, currency, min_cents, effective_from, calibrated_at, calibrated_by)\n`
  sql += `VALUES\n`
  sql += floorsCsv.map(r =>
    `  ('${r.format}', '${r.intrusion_level}', '${r.licence_class}', '${r.currency}', ${r.min_cents}, now(), now(), '${calibratedBy}')`
  ).join(',\n') + ';\n'

  writeFile('supabase/migrations/_pending/<TIMESTAMP>_pricing_seed_v1.sql', sql)
  console.log('Wrote SQL seed to _pending/. Move into supabase/migrations/ when F2 ships.')
}
```

### 10.2 Output location

Generated SQL lands in `supabase/migrations/_pending/` (a directory the converter creates if absent). F2 directive will move it to `supabase/migrations/` with a timestamp prefix at F2 ship time.

The `_pending/` directory is gitignored or kept out of the migration runner's path so accidental untimed-migration application doesn't happen.

### 10.3 Re-run semantics

Re-running the converter with edited CSVs regenerates `_pending/` SQL with `table_version = 1`. **The version bump is manual** — when calibration is recalibrated post-launch, the founder bumps the version constant in the converter script (or passes `--version=2` flag) and the new SQL preserves old rows + adds new rows per F1 §3.3 recalibration cadence.

For v1 seed: `table_version = 1` is the only value the converter produces.

### 10.4 SQL escaping

The `calibration_basis` strings may contain quotes, commas, etc. The converter uses parameterized SQL OR proper SQL escaping (`E'...'` strings with backslash escapes; or `$$...$$` dollar-quoting). The reference implementation must handle:
- Single quotes in basis text (e.g., "fotoQuote's 2024 editorial midpoint")
- Newlines in basis text (rare; escape as `\n`)
- Unicode in basis text (UTF-8 input → UTF-8 output)

## 11. Founder workflow (the actual calibration pass)

```
Phase 1 — Preparation (~30 min)
  1. Read this directive (F1.5) end to end
  2. Read F1 §3 + §4 (table definitions) + brief v3 §4.2.3 (format_defaults adapter)
  3. Open the CSV templates in your editor / spreadsheet of choice
  4. Skim the calibration sources (§8) and pick which 2-3 you'll consult

Phase 2 — Spine cells (~1-2 hr)
  5. Anchor 3-5 spine cells per §6.3:
     - photo / standard / Editorial / mid_pub  (most common cell)
     - photo / standard / Editorial / major_pub  (publication-size scale)
     - photo / standard / Campaign / commercial  (licence-class spread)
     - photo / heavy / Editorial / mid_pub  (intrusion-level spread)
  6. For each spine cell: consult sources; pick value; write calibration_basis

Phase 3 — Derive remaining (~3-5 hr)
  7. Fill remaining photo cells by ratio from the spine
  8. Fill illustration cells (often photo × 0.7-1.2 depending on creator effort)
  9. Fill infographic cells (typically higher than photo for editorial; lower for social)
  10. Fill vector cells (usually below illustration; format simplicity)
  11. Fill video / audio / text cells using format-specific external sources

Phase 4 — Floors (~30 min)
  12. Fill platform_floors_v1_eur.csv per §7.3 (strictly below min defaults for each combo)
  13. Cross-check: every floor cell ≤ min(use_tiers) for that (format, intrusion, licence)

Phase 5 — Validation + seed (~10 min Claude work after CSVs filled)
  14. Run scripts/pricing/csv-to-seed-migration.ts; resolve any validation errors
  15. Review the generated SQL seed
  16. Sanity-check 5-10 random cells against the source: does the row's basis + cents make sense?

Phase 6 — Ratify F1.5 + ship F2 (Claude composes F2 directive after this gate)
  17. Founder reviews the filled CSVs as a docs PR
  18. Approve the calibration; F2 schema migration directive composes against now-known seed values
```

**Estimated total: 6-10 hours of founder time** (spread across days; not contiguous). The bulk is the spine + ratio derivation phase.

## 14. Don't-do list

1. **Don't fill CSVs in F1.5's PR.** This directive ships the templates + tooling; calibration values are Stage B (founder offline work + separate PR).
2. **Don't skip the validation pass.** The converter's validation catches typos, missing rows, out-of-bound values — these are the most common errors when filling a 252-row table by hand.
3. **Don't add multi-currency support here.** EUR-only v1 per F1 §5.2. Multi-currency is per-currency calibration directives (F1.5-USD, F1.5-GBP, etc.).
4. **Don't cite external sources as authoritative in `calibration_basis`.** The column is internal documentation for recalibration teams. Per brief v3 §4.2.4, the engine's runtime basis statement to creators uses Frontfiles-positioned language; the calibration_basis is the founder's note for future-self / future-team.
5. **Don't auto-generate "spine" cell values.** Founder picks the anchors. The directive offers spine-cell candidates as workflow guidance, not as required values.
6. **Don't ship the generated SQL outside `_pending/` until F2.** F2 directive timestamps + moves the seed into `supabase/migrations/`.
7. **Don't bypass the floors cross-check.** Even though it's a warning (not an error), every flagged cell deserves explicit founder review before F2.
8. **Don't add per-format exclusivity rows.** Exclusivity is downstream of base price per `EXCLUSIVE_MULTIPLIERS` + brief v3 §4.6 + F1 §3.1. F1.5's CSVs do NOT have an `exclusivity` column.
9. **Don't introduce floors below the platform's commercial reality.** A `min_cents` of €1.00 effectively disables the floor for that combination. Floors should be set at meaningful values (typical: 30-70% of the corresponding small_pub default, depending on format).
10. **Don't merge F2 before this directive ratifies AND the calibration values are filled.** Both gates required per §13.
