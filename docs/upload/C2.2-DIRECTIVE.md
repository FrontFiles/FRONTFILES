## 1. What this directive is

A focused implementation prompt for C2.2. Lands the visual layer for the asset list — the heart of the new upload UX. C2.1 mounted the shell with placeholder regions; C2.2 replaces the asset-list placeholder with a full density-aware list that auto-switches between four modes based on `assetOrder.length`.

Three artifact groups:

1. **Asset list components** — `AssetList.tsx` (router/container), `AssetRow.tsx` (Linear variant), `AssetRowCompact.tsx` (Compact/Batch/Archive variant), `StoryGroupAccordion.tsx` (Archive cluster shape).
2. **Bulk + filter surfaces** — `BulkOpsBar.tsx`, `FilterBar.tsx`, `AIProposalBanner.tsx`.
3. **New mock scenarios + selector additions** — 3 Archive-scale fixtures in `v2-mock-scenarios.ts` (150 / 500 / 1500 assets), plus a new `getFilteredSortedSearchedAssets` selector in `upload-selectors.ts` since the existing `getFilteredAssets` lives in dormant `v2-state.ts` and depends on V2State.

This directive is build-governing for C2.2. No actual region behavior beyond what UX-SPEC-V3 specifies. C2.3 (side detail panel), C2.4 (commit flow), C2.5 (AI proposal acceptance interactions) compose separately.

### 3.2 `AssetList.tsx` (new) — density-aware container

Reads `state.assetOrder.length`, derives density via `densityForCount`, renders the appropriate variant:

| Density | Renders |
|---|---|
| `linear` | Stack of `AssetRow` (full per-row detail) |
| `compact` | Stack of `AssetRowCompact` (collapsed rows) |
| `batch` | `AssetRowCompact` + sticky `BulkOpsBar` (visible) + `FilterBar` + AI cluster proposal banners |
| `archive` | `StoryGroupAccordion` (per cluster) + ungrouped section + `BulkOpsBar` + `FilterBar` + accordion controls — UNLESS `state.ui.flatListOverride` is on, in which case render flat Compact-style list |

Uses `react-window` for virtualization in Compact/Batch/Archive (Linear is small-N, no virtualization needed).

Imports the new `getFilteredSortedSearchedAssets` selector from `upload-selectors.ts` to apply current filter/sort/search before rendering.

### 3.3 `AssetRow.tsx` (new) — Linear variant per UX-SPEC-V3 §3.1

Full per-row detail. Editable inline:
- Thumbnail (~96×96px)
- Filename (small caps, read-only)
- Title input (required)
- Caption input (with AI ghost-text + ✓ accept)
- Tags chip input (with AI ghost chips)
- Price currency input (with engine ghost recommendation + "Why?" link + ✓ accept)
- Privacy dropdown (Private/Restricted/Public)
- Licences multi-select dropdown
- Geo input
- Exclude affordance (top-right ✕ Exclude toggle)

Field validation per spec §3.3: title required; price/licences required if PUBLIC/RESTRICTED.

AI proposal accept actions dispatch via `useUploadContext`. Per IP-5 (selector-derived acceptance), accepting a proposal dispatches `UPDATE_ASSET_FIELD` (auto-commit semantics from spec §9.1) — NOT a separate accept action.

### 3.4 `AssetRowCompact.tsx` (new) — Compact/Batch/Archive variant per spec §4.1

~64px tall row:
- Thumbnail (~48×48px) with multi-select clickable left edge (~24px wide, cursor:pointer, 4px Frontfiles-blue border when selected)
- Filename + format chip
- Caption preview (single line, truncates with ellipsis; italic if AI-suggested unaccepted)
- Inline price with ✓ accept
- Status chip cluster (right side, max 3 visible, tooltip for more)

Click anywhere on row (except the left-edge select area) opens side detail panel via `OPEN_SIDE_PANEL` (panel itself is C2.3).

### 3.5 `StoryGroupAccordion.tsx` (new) — Archive cluster shape per spec §6.1

Cluster header:
- Expand/collapse toggle (▶/▼)
- Cluster name (inline-editable; click to rename)
- Asset count + ready count + status dot summary
- Cluster-level bulk action buttons: "Accept all suggestions", "Bulk-edit caption", "Set price"

Cluster body:
- Virtualized scroll list of `AssetRowCompact` for assets in this cluster
- Reordering via drag-drop into other clusters (deferred to C3 — for C2.2, render-only)

"Ungrouped" section always at bottom — assets where `storyGroupId === null`.

Initial accordion state (per IP-3 below): first cluster expanded, others collapsed.

### 3.6 `BulkOpsBar.tsx` (new) — per spec §4.2

Renders when:
- `state.ui.bulkOpsBarOpen === true` (Compact mode toggle), OR
- `densityForCount(state.assetOrder.length)` is `'batch'` or `'archive'` (auto-shown)

Layout:
```
┌──────────────────────────────────────────────────────────────┐
│  3 selected  •  Set price: [____]  Set privacy: [Private ▼] │
│  Apply to selected: [Caption] [Tags] [Geo]    [Clear]        │
└──────────────────────────────────────────────────────────────┘
```

Actions dispatch via `useUploadContext`:
- "Set price" → `BULK_UPDATE_FIELD` with `field='price'` for selected assets
- "Set privacy" → `BULK_UPDATE_FIELD` with `field='privacy'`
- "Apply to selected: Caption/Tags/Geo" → expand inline popover, then `BULK_UPDATE_FIELD`
- "Clear" → `DESELECT_ALL_ASSETS`

Selected count = `state.ui.selectedAssetIds.length`.

### 3.7 `FilterBar.tsx` (new) — per spec §5.1

Renders in Batch and Archive modes only. Layout:

```
┌──────────────────────────────────────────────────────────────┐
│  Filter: [All] [Needs info] [Duplicates] [Ready] [Excluded] │
│  Sort: [File order ▼]    Search: [____________________]      │
└──────────────────────────────────────────────────────────────┘
```

Filter chips reflect §10 exception categories — exclusive (radio-style), per IP-2 below.

Filter chip → `SET_FILTER_PRESET` action.
Sort dropdown (file order / filename A→Z / file size / format) → `SET_SORT` action.
Search input (debounced 200ms per IP-4) → `SET_SEARCH_QUERY` action.

### 3.8 `AIProposalBanner.tsx` (new) — per spec §5.2

Renders when:
- `densityForCount(state.assetOrder.length)` is `'batch'` or `'archive'`, AND
- `state.aiClusterProposals` has entries with `status === 'pending'`

Each banner:
```
┌──────────────────────────────────────────────────────────────┐
│  ◇ 5 assets appear to be from one shoot — accept as a group?│
│  [Accept] [Dismiss] [See details]                            │
└──────────────────────────────────────────────────────────────┘
```

4px Frontfiles-blue left edge accent.

Actions:
- "Accept" → `ACCEPT_AI_CLUSTER_PROPOSAL` (creates Story group)
- "Dismiss" → `DISMISS_AI_CLUSTER_PROPOSAL` (banner hides; doesn't recur this session)
- "See details" → expand inline (no separate action; component-local UI state)

Multiple banners stack independently.

### 3.9 New selector — `getFilteredSortedSearchedAssets`

Add to `src/lib/upload/upload-selectors.ts` — operates on the narrow input view. Replaces the V2-coupled `getFilteredAssets` in `v2-state.ts` (which depends on `V2UIState` shape and is dormant).

Signature:

```typescript
export interface FilterableView extends AssetsView {
  filter: V3UIState['filter']
  searchQuery: string
  sortField: V3UIState['sortField']
  sortDirection: V3UIState['sortDirection']
}

export function getFilteredSortedSearchedAssets(view: FilterableView): V2Asset[]
```

Implementation mirrors `getFilteredAssets` from v2-state.ts — same filter presets, same sort logic, same search semantics. The narrow input shape decouples from V2UIState/V3UIState directly.

V3State satisfies `FilterableView` via structural typing: `{ assetsById, assetOrder, filter: ui.filter, searchQuery: ui.searchQuery, sortField: ui.sortField, sortDirection: ui.sortDirection }` — wrapper at the call site spreads from V3State.ui.

### 3.10 New action — `BULK_UPDATE_FIELD` (already exists; verify signature)

Already in V3Action union from C2.1. Verify the BulkOpsBar dispatches it correctly with the right shape.

### 3.11 Reducer addition — `densityForCount` integration

No new reducer action. Density is computed at render time per IP-6 of C2-PLAN composition. AssetList reads `state.assetOrder.length` and derives density via `densityForCount`.

### 3.12 UploadShell.tsx update

## 4. Acceptance criteria

- [ ] 1–5 files → Linear mode: full per-row `AssetRow` rendered, no virtualization, no FilterBar, no BulkOpsBar
- [ ] 6–19 files → Compact mode: `AssetRowCompact`, virtualized; BulkOpsBar HIDDEN by default (toggleable via the existing `TOGGLE_BULK_OPS_BAR` action)
- [ ] 20–99 files → Batch mode: `AssetRowCompact`, virtualized; BulkOpsBar VISIBLE by default; FilterBar present
- [ ] 100+ files → Archive mode: `StoryGroupAccordion` per cluster + ungrouped section + BulkOpsBar + FilterBar
- [ ] Adding a 6th file mid-session auto-switches Linear → Compact (no manual SET_DENSITY action)
- [ ] "View as flat list" toggle in Archive mode dispatches `TOGGLE_FLAT_LIST_OVERRIDE`; flips to Compact-style flat list
- [ ] Multi-select via row left-edge click works (Compact+); selected rows show 4px Frontfiles-blue left border
- [ ] Filter chips dispatch `SET_FILTER_PRESET`; selected chip visually distinct
- [ ] Sort dropdown dispatches `SET_SORT`; sort applied
- [ ] Search input (debounced 200ms) dispatches `SET_SEARCH_QUERY`; assets filtered
- [ ] BulkOpsBar "Set price" dispatches `BULK_UPDATE_FIELD` with `field='price'`
- [ ] BulkOpsBar "Apply Caption/Tags/Geo" dispatches `BULK_UPDATE_FIELD`
- [ ] BulkOpsBar "Clear" dispatches `DESELECT_ALL_ASSETS`
- [ ] AI cluster proposal banners render when `aiClusterProposals` has pending entries (Batch + Archive); accept/dismiss actions dispatched correctly
- [ ] Archive accordion: first cluster expanded, others collapsed initially; click cluster header dispatches `TOGGLE_CLUSTER_EXPANDED`
- [ ] Archive cluster name editable inline (click → contenteditable → blur dispatches `RENAME_STORY_GROUP`)
- [ ] 3 new mock scenarios load via existing scenario-registry path
- [ ] Parity test extended to cover 3 new fixtures (existing test pattern)

## 7. Don't-do list (specific to C2.2)

1. **Don't dispatch `SET_DENSITY`.** No such action exists in V3. Density is computed at render time from `state.assetOrder.length`.
2. **Don't import `getFilteredAssets` from `v2-state.ts`.** The new `getFilteredSortedSearchedAssets` in `upload-selectors.ts` is canonical for V3. The dormant V2 version stays in v2-state.ts unchanged.
3. **Don't allow bulk-accept of price.** UI must NEVER expose a "Bulk accept price" affordance. The reducer's `bulk_accept_price_forbidden` throw is the safety net; the UI is the first line.
4. **Don't render Linear mode with virtualization.** It's small-N (≤5); virtualization is overhead.
5. **Don't render the side detail panel from C2.2.** That's C2.3. AssetRowCompact dispatches `OPEN_SIDE_PANEL`; the panel rendering itself comes later.
6. **Don't render the commit summary or commit progress from C2.2.** That's C2.4.
7. **Don't render AI proposal acceptance UI inside `AssetRow` interactions.** That's C2.5 (the visual ✓ button + acceptance flows are scoped there). C2.2 renders the AI ghost text per spec §3.2 but the interaction wiring is C2.5.
8. **Don't auto-show BulkOpsBar in Compact mode.** It's HIDDEN by default in Compact (toggleable). It's auto-shown only in Batch + Archive.
9. **Don't introduce new V3 actions.** All needed actions exist in C2.1's V3Action union (`SET_FILTER_PRESET`, `SET_SORT`, `SET_SEARCH_QUERY`, `BULK_UPDATE_FIELD`, `DESELECT_ALL_ASSETS`, `TOGGLE_CLUSTER_EXPANDED`, `RENAME_STORY_GROUP`, `TOGGLE_BULK_OPS_BAR`, `TOGGLE_FLAT_LIST_OVERRIDE`, `OPEN_SIDE_PANEL`, `ACCEPT_AI_CLUSTER_PROPOSAL`, `DISMISS_AI_CLUSTER_PROPOSAL`). If C2.2 needs a new action, surface it as a HALT — don't add silently.
10. **Don't extend `v2-state.ts`.** All selector additions go in `upload-selectors.ts`.
