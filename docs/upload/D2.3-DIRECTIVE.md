## 0. Context

D2.1 landed the shell with placeholder pane content. The center pane currently shows `Center pane — placeholder for D2.3 (layout: workspace, zoom: 3)`. D2.3 fills it.

This is **the surface that gives the V4 product its editorial character.** The C2 work read as control-heavy because the center was a row-list with chips. The V4 center is a visual contact sheet — assets dominate, chrome is contextual, AI furniture is invisible (lives only in the right rail).

D2.3 ships the contact sheet structurally complete: card grid, virtualization, zoom slider, filter chips, search, count footer, click-driven selection. **DnD source wiring is explicitly deferred to D2.2** so it lands alongside the left-rail receivers (no dead-code interim where drag works but has nowhere to drop). Sortable-within-story reordering rides the same DnD ratification.

After D2.3 lands, the visual product reads as the right product even though left/right rails are still placeholder shapes. D2.4 fills the right rail; D2.2 wires the left rail + DnD; D2.5 adds the contextual action bar; D2.6 adds compare; D2.7 wires file ingestion.

## 4. Implementation Points (ratify before implementation)

### IPD3-1 — DnD scope confirmation
- **(a)** Defer DnD entirely to D2.2 (sources land with receivers; no dead-code interim). Recommended.
- (b) Ship DnD source wiring in D2.3; D2.2 adds receivers later (interim drag has nowhere to drop).
- (c) Install `@dnd-kit` infra in D2.3 but no source wiring (just DragContext setup).
- **Default: (a).**

### IPD3-2 — Sortable reorder timing
- **(a)** Defer to D2.2 (rides DnD infra).
- (b) Ship as a separate non-DnD reorder UI in D2.3 (up/down arrow buttons on cards in story-filtered view).
- **Default: (a).**

### IPD3-3 — Double-click behavior in D2.3
- **(a)** Same as single-click (selection follows). The double-click → inspector full-bleed affordance lands in D2.4 once the real inspector exists. Recommended; no fake feature.
- (b) Wire the double-click to a no-op handler with a TODO comment.
- (c) Skip double-click entirely; selection only via single-click.
- **Default: (a).**

### IPD3-4 — Status dot mapping
Per L8, five colors map to states. Resolution rules per asset:
- Excluded → red
- `analysisStatus !== 'complete'` → gray (processing)
- `duplicateStatus === 'likely_duplicate'` → orange
- Has blocking exception (filtered for needs_story per UX-BRIEF v3 §4.5) → yellow
- Has only advisory (low_confidence / provenance_pending) → gray (low confidence)
- Otherwise → green (ready)
- **(a)** Above resolution order (excluded > processing > duplicate > blocking > advisory > ready). Recommended.
- (b) Different priority (e.g., blocking > duplicate).
- **Default: (a).**

### IPD3-5 — Story badge content
- **(a)** Show the story name (truncated to ~16 chars) when `asset.storyGroupId !== null`. Recommended.
- (b) Show only a small icon (📁 or similar) — minimal.
- (c) No badge — assignment shown only via filter chips.
- **Default: (a).**

### IPD3-6 — Search affordance
- **(a)** Magnifying-glass icon at the right side of filter chips row; click reveals an inline input that takes focus. Esc dismisses, blank value clears. Debounced 200ms (reuses C2 FilterBar pattern). Recommended — keeps the chip row clean by default.
- (b) Always-visible search input alongside filter chips.
- **Default: (a).**

### IPD3-7 — Count footer formula
- **(a)** `{visibleCount} of {includedCount} · {readyCount} ready` — visible = post-filter; included = excluded-aware count. Recommended; matches spec §3.6.
- (b) Just `{visibleCount} ASSETS — {readyCount} READY` (mirrors C2 cluster header style).
- **Default: (a).**

### IPD3-8 — AIProposalBanner mount position
- **(a)** Above filter chips, top of CenterPane. Per L7. Recommended.
- (b) Above the contact sheet but below filter chips.
- (c) Floating overlay at top.
- **Default: (a).**

### IPD3-9 — ContactSheetCard image source
- **(a)** Prefer `asset.thumbnailRef` (URL string) → object-URL of `asset.file` (with proper revoke on unmount/asset-change) → brutalist placeholder (filename + format watermark). Recommended; same pattern as C2 SideDetailPanel.
- (b) Always use placeholder until D2.7 wires real upload (no real thumbnails until then).
- **Default: (a).**

### IPD3-10 — Test depth for D2.3
- **(a)** No new reducer or selector tests (no new actions; no new selectors). Visual smoke is the gate. Same posture as C2.3-C2.6 + D2.1 component-only phases.
- (b) Extract status-dot-resolution as a tiny helper + add a few unit tests for the resolution priority.
- **Default: (a).** (b) is a viable enhancement when the helper is ready for reuse — premature in D2.3.

### IPD3-11 — Selection model edge cases
- Range-select (Shift-click) needs an "anchor" — the last single-click target. Where does this live?
  - **(a)** Local component state in `ContactSheet.tsx`. Recommended; ephemeral, doesn't belong in V3State.
  - (b) New V3UIState field `selectionAnchorId`.
  - **Default: (a).**

### IPD3-12 — Click target on card
- **(a)** Whole card body is the click target. The selection checkbox in the top-left corner is a separate hit area (toggle-only). Story badge in top-right is non-clickable (display only). Recommended.
- (b) Filename area only.
- **Default: (a).**

## 5. Acceptance criteria

- `?scenario=clean_single_story` → CenterPane renders 7 ContactSheetCards in a grid at zoom 3 (default)
- `?scenario=archive_150_mixed` → 150 cards rendered (virtualized via react-window)
- Filter chips at top: All (selected by default), Needs info, Ready, Duplicates, Excluded — exclusive selection
- Search magnifying-glass icon → click reveals input → 200ms debounced → filters cards
- Zoom slider at bottom-left: 5 discrete steps; clicking changes `state.ui.contactSheetZoom`; grid columns reflow
- Count footer at bottom-right: e.g., `7 of 7 · 5 ready`
- ContactSheetCard renders: thumbnail (or placeholder) + filename overlay on hover + selection checkbox top-left (visible on hover OR when selected) + story badge top-right (only when assigned) + status dot bottom-right
- Status dot color follows IPD3-4 resolution priority
- Single-click on card body → replaces selection (right rail placeholder mounts via D2.1 wiring)
- Cmd-click → toggles in selection
- Shift-click from anchor → range-select inclusive
- Double-click → currently same as single-click; D2.4 will add full-bleed
- AIProposalBanner appears at top of CenterPane when `aiClusterProposals` has pending entries (use `?scenario=archive_150_mixed&seedBanners=1` to verify)
- All 172 existing tests still pass (regression gate)
- No new tests required (per IPD3-10 = a)
