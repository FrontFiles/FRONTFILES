## 0. Context

D2.1 stood up the shell with a placeholder left rail (mounted at `assetOrder.length > 0` per the corrected spec §2.0.1). D2.3 made the contact sheet selectable. D2.4 made the right rail editable. The organization loop is missing one thing: **a way to move assets between stories visually.** D2.2 closes that loop.

Per IPD3-1 (ratified at D2.3), DnD source + receiver wiring all land here so there's no dead-code interim where drag works on cards but has nowhere to drop.

D2.2 ships:

1. **Left rail content** — Stories + Unassigned + + New affordance
2. **DnD infra** — `@dnd-kit/core` + `@dnd-kit/sortable` install + DragContext at the shell root
3. **DnD source wiring on `ContactSheetCard`** (the C2.3 file gets a small augmentation to become draggable)
4. **Drop targets** — story headers + Unassigned bucket + cover area within story header
5. **Sortable within story** — when the filter narrows to a single `storyGroupId`, contact-sheet cards become sortable; reorder dispatches `REORDER_ASSETS_IN_STORY`
6. **Filter-by-story flow** — clicking a story header in the left rail dispatches `SET_FILTER { storyGroupId: g.id }`; the contact sheet narrows
7. **Cover thumbnail in story header** — uses the D2.1 `getStoryCover` selector

After D2.2 lands, the organization loop is functional end-to-end: drag a card from the contact sheet onto a story header → asset assigned. Drag onto the cover area → set as cover. Click a story → contact sheet filters to that story → drag-reorder to set sequence.

## 4. Implementation Points (ratify before implementation)

### IPD2-1 — Filter-by-story mechanism
- **(a)** Reuse existing `SET_FILTER { storyGroupId }`. The V2Filter.storyGroupId field already exists in the type. Verify the selector applies it (it does, via getFilteredSortedSearchedAssets). Recommended.
- (b) Add a new "viewMode" concept (e.g., `'all-flat' | 'in-story-{id}'`). Bigger reducer change; not justified.
- **Default: (a).**

### IPD2-2 — Cover thumbnail size in story header
Per L9, 16:9. Size:
- (a) 48×27 — very compact
- **(b) 64×36** — readable but unobtrusive. Recommended.
- (c) 96×54 — eats more vertical space; story rows get taller
- **Default: (b).**

### IPD2-3 — Cover-area drop sub-target
Per L4 + UX-SPEC-V4 §8.2: drop on cover area = MOVE + SET_STORY_COVER. Implementation:
- **(a)** Two `useDroppable` calls in LeftRailStoryHeader — one on the header body (id `story-{id}-body`) and one on the cover thumbnail (id `story-{id}-cover`). Drag-end handler in UploadShellV4 routes by id. Recommended.
- (b) One drop zone with internal hit-testing on drag-end (compute relative position).
- **Default: (a).** Cleaner separation; @dnd-kit's accessibility tooling treats them as distinct zones.

### IPD2-4 — + New story affordance shape
- **(a)** Button at bottom of rail. Click → swaps to inline `<input>` autofocused. Enter creates via `CREATE_STORY_GROUP`; Esc cancels; blur with empty value cancels. Recommended.
- (b) Always-visible input at bottom.
- (c) Modal dialog.
- **Default: (a).**

### IPD2-5 — Sortable-within-story trigger
- **(a)** Active when `state.ui.filter.storyGroupId !== null` AND visible cards are all in that story. Recommended; matches spec §8.2.
- (b) Always sortable on the contact sheet (any zoom, any filter).
- **Default: (a).** Reordering "all visible" doesn't make sense when the visible set spans multiple stories.

### IPD2-6 — Multi-select drag behavior
- **(a)** When `selectedAssetIds.length > 1` AND user drags a card that's in the selection, all selected assets follow. Per L11. Visual: card stack behind cursor + count badge. On drop, dispatch the action per asset in a loop. Recommended.
- (b) Single-asset drag only — multi-select drag deferred.
- **Default: (a).** Spec §8.3 calls for it; @dnd-kit supports it via the DragOverlay pattern.

### IPD2-7 — DragOverlay visual
- **(a)** Render a copy of the dragged card (or stack visual for multi-select) at the cursor via `<DragOverlay>`. Recommended.
- (b) No overlay — let the original card move.
- **Default: (a).** Smoother, accessibility-friendlier.

### IPD2-8 — Drop-target visual feedback
- **(a)** When a drag is in progress AND hovering a valid drop target, target border becomes blue-600 + small "Drop to assign" or "Drop to set as cover" inline label. Recommended.
- (b) Border-only highlight, no label.
- **Default: (a).** Discoverability matters for first-time users.

### IPD2-9 — `@dnd-kit` version
- **(a)** Pin to `^6.0.0` (stable). Recommended.
- (b) Use `^7.0.0` (beta).
- **Default: (a).**

### IPD2-10 — Test depth
- **(a)** No new tests — DnD is hard to test in vitest without DOM (still no jsdom infra). Visual smoke is the gate. Recommended.
- (b) Add reducer test for new dispatches' shapes (most already covered).
- **Default: (a).** D2.2 invents zero new actions; the existing reducer tests cover the action shapes that DnD will dispatch.

### IPD2-11 — `+ Add files` button click handler
- **(a)** No-op stub with `// TODO(D2.7)` comment. Recommended for D2.2 minimum scope.
- (b) Wire it to `document.querySelector('input[type=file]').click()` with a hidden input — premature; D2.7 will own the file ingestion path properly.
- **Default: (a).**

### IPD2-12 — Settings cog click handler
- **(a)** No-op stub with `// TODO(D2.7)`. Same as IPD2-11. Recommended.
- (b) Open an empty popover.
- **Default: (a).**

### IPD2-13 — Story click filters AND deselects?
When the user clicks a story header in the left rail, the filter narrows. Should the right rail (single-asset selection) clear too?
- **(a)** Yes — also dispatch `DESELECT_ALL_ASSETS`. Story navigation is a different intent; clearing selection prevents the inspector from showing an asset that may no longer be in the visible set. Recommended.
- (b) No — keep the current selection. If the asset isn't in the new filtered view, it's hidden but still selected.
- **Default: (a).**

### IPD2-14 — Click outside the left rail OR clicking "All" filter — clears storyGroupId filter?
- **(a)** Clicking the "All" chip in CenterPane filter chips clears `storyGroupId` (and resets preset to 'all'). Already happens via existing SET_FILTER_PRESET handler if it overrides storyGroupId. Verify.
- (b) Need a new action.
- **Default: (a).** Verify reducer behavior; if SET_FILTER_PRESET doesn't already clear storyGroupId, augment SET_FILTER_PRESET handler to do so.

## 5. Acceptance criteria

- Left rail renders with: header (+ Add files, cog), Stories list (with cover thumbs at 16:9 64×36), Unassigned bucket, + New story affordance
- Cover thumbnail in story header reads first asset by default; updates when SET_STORY_COVER dispatched
- Click a story header → contact sheet filters to that story (storyGroupId filter applied); selection cleared
- Click Unassigned bucket → contact sheet shows only ungrouped assets
- Click "All" filter chip in CenterPane → clears storyGroupId filter; full asset set visible
- + New story → inline input → Enter creates story; Esc cancels
- Drag a card from contact sheet:
  - Onto a story header body → asset assigned to that story (`MOVE_ASSET_TO_CLUSTER`)
  - Onto a story header cover area → asset assigned AND set as cover (`MOVE_ASSET_TO_CLUSTER` + `SET_STORY_COVER`)
  - Onto Unassigned bucket → asset unassigned (`MOVE_ASSET_TO_UNGROUPED`)
- Multi-select drag (`selectedAssetIds.length > 1`):
  - Card stack visual behind cursor
  - Drop applies the action to ALL selected assets
- When filter narrows to a single story → contact sheet cards become sortable; drag-reorder dispatches `REORDER_ASSETS_IN_STORY` with the new sequence
- Left rail collapse toggle works (`TOGGLE_LEFT_RAIL_COLLAPSED`); collapsed state shows a 40px icon strip
- DragOverlay smoothly tracks the cursor; respects `prefers-reduced-motion`
- Drop targets highlight with blue-600 border + label on hover during drag
- All 172 existing tests still pass (regression gate)
