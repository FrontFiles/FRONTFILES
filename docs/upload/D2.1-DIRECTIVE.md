## 4. Reducer + data model deltas (the contract surface)

### 4.1 New V3Action types

Six additions to `V3Action` union:

```typescript
// Story cover + sequencing (per spec §7)
| { type: 'SET_STORY_COVER'; storyGroupId: string; assetId: string | null }
| { type: 'REORDER_ASSETS_IN_STORY'; storyGroupId: string; sequence: string[] }

// Contact sheet zoom (per spec §3.5)
| { type: 'SET_CONTACT_SHEET_ZOOM'; zoom: 1 | 2 | 3 | 4 | 5 }

// Left rail collapse (per spec §2.1)
| { type: 'TOGGLE_LEFT_RAIL_COLLAPSED' }

// Compare mode (per spec §10)
| { type: 'ENTER_COMPARE_MODE'; assetIds: string[] }
| { type: 'EXIT_COMPARE_MODE' }
```

### 4.2 Reducer handler contracts

Strict guards. Phase-specific error messages so failures point at the broken contract.

| Action | Validation | Mutation |
|---|---|---|
| `SET_STORY_COVER` | Throws `set_story_cover_invalid_group` if storyGroupId not in `state.storyGroupsById`. If `assetId` is non-null, throws `set_story_cover_invalid_asset` if not in `state.assetsById`. `assetId === null` clears the cover (auto-fallback at render). | Updates `state.storyGroupsById[storyGroupId].coverAssetId` |
| `REORDER_ASSETS_IN_STORY` | Throws `reorder_assets_invalid_group` if storyGroupId not in `state.storyGroupsById`. Throws `reorder_assets_set_mismatch` if the sequence array doesn't match the existing membership (set-equality check). | Replaces `state.storyGroupsById[storyGroupId].sequence` with the new array. Does NOT change `coverAssetId`. |
| `SET_CONTACT_SHEET_ZOOM` | Throws `set_contact_sheet_zoom_invalid` if zoom not in {1,2,3,4,5}. | Updates `state.ui.contactSheetZoom` |
| `TOGGLE_LEFT_RAIL_COLLAPSED` | None. | Flips `state.ui.leftRailCollapsed` |
| `ENTER_COMPARE_MODE` | Per IPD1-4 default = (a) strict 2-only. Throws `compare_invalid_count` if `assetIds.length !== 2`. Throws `compare_invalid_asset` if any id not in `state.assetsById`. | Sets `state.ui.compareAssetIds = assetIds`. Does NOT change `selectedAssetIds`. |
| `EXIT_COMPARE_MODE` | None. Idempotent. | Sets `state.ui.compareAssetIds = []` |

## 5. Hydration changes

### 5.1 V2StoryGroup field defaults

`hydrateV3FromV2State` in `v3-hydration.ts`:

```typescript
const storyGroupsById = Object.fromEntries(
  Object.entries(v2State.storyGroupsById).map(([id, g]) => [
    id,
    {
      ...g,
      coverAssetId: g.coverAssetId ?? null,                // additive default
      sequence: g.sequence ?? [...g.proposedAssetIds],     // initial sequence = proposed order
    },
  ]),
)
```

Existing fixtures parse cleanly. Sequence diverges from `proposedAssetIds` as soon as the user reorders (D2.3).

### 5.2 AI auto-accept sweep

Per spec §11.1, runs after the V2State→V3State bridge. For each asset in `assetsById`:

```typescript
const AI_AUTO_ACCEPT_THRESHOLD = 0.85  // per IPV4-5 default; constant in v3-hydration.ts

if (asset.proposal && asset.proposal.confidence >= AI_AUTO_ACCEPT_THRESHOLD) {
  // Per IPD1-3 default = (a): caption + tags + geography only. NEVER price (L3 + spec §9.2).
  // NEVER title (creators author titles personally per IPD1-3 rationale).
  if (!asset.editable.description && asset.proposal.description) {
    asset.editable.description = asset.proposal.description
  }
  if (asset.editable.tags.length === 0 && asset.proposal.tags?.length) {
    asset.editable.tags = [...asset.proposal.tags]
  }
  if (asset.editable.geography.length === 0 && asset.proposal.geography?.length) {
    asset.editable.geography = [...asset.proposal.geography]
  }
}
```

### 5.3 Implications

After hydration, asset.editable contains AI-suggested values for high-confidence proposals. The contact sheet ContactSheetCard (D2.3) shows no "ghost text" for these — they read as user-authored. The right rail inspector (D2.4) only shows ghost + ✓ + ↻ for low-confidence (<0.85) fields.

For the `archive_150_mixed` fixture: most assets have moderate confidence; some auto-accept. For `clean_single_story`: most are high confidence; nearly all auto-accept. Visual smoke at D2.1 may not reveal much (placeholders only); D2.3 will show the difference.

### 5.4 Tests

The parity test suite (`v3-state-parity.test.ts`) currently asserts that after hydration, `editable.description === ''` for assets with proposals. **This will break** because high-confidence proposals now auto-accept into `editable.description`. **Update the parity tests** to reflect the new hydration contract: assert `editable.description === asset.proposal.description` for high-confidence; `editable.description === ''` for low-confidence.

This is an explicit spec exception per L3 + spec §11.1; documented in the test comments.

- **(a)** Strict: throws `compare_invalid_count` if `assetIds.length !== 2`. Matches IPV4-3 lock. Recommended.

- **(a)** Returns `'empty' | 'workspace' | 'comparing'` — string union. Matches spec §2. Recommended.

- **Default: (a).** Components derive pane visibility from `state.assetOrder.length`, `state.ui.leftRailCollapsed`, `state.ui.selectedAssetIds.length` separately. The selector returns one fact: layout state.
