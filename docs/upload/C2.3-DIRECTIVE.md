## 0. Context

C2.1 stood up the V3 reducer + page surface with placeholders. C2.2 made the asset list live across four density modes. The Region 3 placeholder in `UploadShell.tsx` (lines 75–82) currently renders a string when `state.ui.sidePanelOpenAssetId` is set — but nothing dispatches `OPEN_SIDE_PANEL`. C2.3 builds the real panel and wires the click → open → edit → close loop end-to-end.

The reducer surface for the panel already exists: `OPEN_SIDE_PANEL { assetId }`, `CLOSE_SIDE_PANEL`, `NAVIGATE_SIDE_PANEL { direction }`. Editing actions also exist (`UPDATE_ASSET_FIELD { assetId, field, value }` is the generic per-field dispatcher; `RESOLVE_DUPLICATE` handles the duplicate-resolver flow; `RESOLVE_CONFLICT` handles EXIF-vs-manual conflicts). C2.3 invents NO new action types — if it needs one, that's a HALT-and-surface signal.

## 4. Implementation Points (ratify before implementation)

The implementing Claude must HALT and surface if any of these change shape mid-build. Defaults are recommended; founder ratification = the directive locks them.

### IPIII-1 — Click target on Compact rows
Spec §6.2 line 181 says "Click anywhere on row: opens side detail panel (§7) showing all fields like Linear mode". So whole-row click. But `AssetRowCompact` has interactive sub-controls (the chips, the future ✓ accept icons in C2.5).
- **(a) Whole row clickable except interactive controls** — chip buttons and any inner button use `e.stopPropagation()` so they don't bubble. Recommended.
- (b) Filename column only.
- (c) Dedicated chevron icon on the right edge.
- **Default:** (a)

### IPIII-2 — Click target on Archive cluster rows
Same question for assets rendered inside expanded `StoryGroupAccordion` bodies.
- **(a) Same as IPIII-1** (whole row except inner controls). Recommended for consistency.
- (b) Different — accordion body uses a smaller hit area to preserve cluster-level interactions.
- **Default:** (a)

### IPIII-3 — Side panel mount when a Linear-mode asset is open via legacy state
The reducer accepts `OPEN_SIDE_PANEL { assetId }` for any asset id, regardless of density. If the user starts in Compact, opens the panel for asset `x`, then deletes enough assets to drop into Linear mode (count < 6), what happens to the open panel?
- **(a) Panel auto-closes on density transition into Linear** — defensive: dispatch `CLOSE_SIDE_PANEL` from a `useEffect` watching `density === 'linear' && sidePanelOpenAssetId !== null`. Recommended; respects L4.
- (b) Panel stays open, ignoring the Linear-no-panel rule.
- (c) Panel stays open, but Linear hides itself.
- **Default:** (a)

### IPIII-4 — Bulk selection coexistence
If `selectedAssetIds.length > 1` (BulkOpsBar visible) and the user clicks a single row:
- **(a) Panel opens for that one asset; bulk selection persists in background** — never block primary interactions. Recommended.
- (b) Panel opens but reflects "5 selected" (panel becomes a bulk-edit form).
- (c) Click is ignored when bulk mode is active.
- **Default:** (a). Bulk-edit panel mode is explicitly out of scope per §1.2.

### IPIII-5 — Stale assetId guard semantics
If the open asset id is removed from `state.assetsById` (e.g., a hypothetical future delete action) or filtered out (it still exists but is hidden by the active filter):
- **(a) Asset removed from map → auto-close.** Asset filtered out → panel STAYS OPEN (filter is a view concern, not an existence concern). Recommended.
- (b) Either condition closes.
- (c) Neither condition closes; component renders a "no asset" state.
- **Default:** (a). Implement the auto-close as a `useEffect` inside `SideDetailPanel` watching `assetsById[sidePanelOpenAssetId]`.

### IPIII-6 — Exceptions section depth
The exceptions section lists active blocking exceptions for the open asset. `getAssetExceptions` returns typed exceptions including `needs_price`, `needs_caption`, `needs_privacy`, `metadata_conflict`, `duplicate`, etc.
- **(a) List ALL exceptions verbatim.** Each gets a one-line label + (where applicable) an inline resolver. `duplicate` → `<DuplicateResolver />`. `metadata_conflict` → conflict resolver UI dispatching `RESOLVE_CONFLICT`. Other blocking exceptions render label only (the editable field above is the resolver).
- (b) Group exceptions by severity.
- (c) Show only the count + a "view all" expand.
- **Default:** (a). Drives a clean read with minimal indirection.

### IPIII-7 — AI Proposal Detail section depth in C2.3
Spec §7.1 shows this as a collapsible section with three rows: caption rationale, price basis, tag confidence. The actual basis content + accept-from-here behavior is C2.5 scope.
- **(a) Render an empty collapsible shell with a `// TODO: C2.5` placeholder body** — preserves layout, no fake content. Recommended.
- (b) Skip the section entirely until C2.5 — risk: spec §7.1 anatomy isn't visible yet, harder for visual QA.
- (c) Stub each of the three rows with proposal text from `state.assetsById[id].proposal` (read-only) but no accept actions.
- **Default:** (a).

### IPIII-8 — Editable field shapes
Spec §7.1 lists Title / Caption / Tags / Price / Privacy / Licences / Geo. Implementation choice per field:
- Title, Caption — `<input>` / `<textarea>`. Caption is multiline.
- Tags — comma-separated `<input>` for V1 (a chip-array editor is overkill for C2.3).
- Price — number `<input>` with EUR suffix; AI ghost text rendered as `placeholder` for unaccepted suggestions (visual-only — accept icon is C2.5).
- Privacy — `<select>` over `'public' | 'private' | 'sensitive'` (verify enum vs `V2Asset.privacy`).
- Licences — `<select>` over license enum (verify against `V2Defaults`).
- Geo — two `<input type="number">` (lat, long) + an "Use file EXIF" button if `extractedMetadata.latitude` / `longitude` exist (dispatches `RESOLVE_CONFLICT` or `UPDATE_ASSET_FIELD` directly).

Confirm shape, or override individual rows. **Default:** as listed.

### IPIII-9 — Test depth
Component test file `__tests__/SideDetailPanel.test.tsx` — yes/no?
- **(a) Yes** — six cases: renders correct asset by id; Esc closes; ✕ closes; J/K navigates within filter; field edit dispatches `UPDATE_ASSET_FIELD` with correct payload; stale-id auto-close. Recommended.
- (b) No — rely on reducer parity tests + visual smoke. Lighter, but the click→dispatch wiring goes untested.
- **Default:** (a). Keeps the dispatch contract verified.

### IPIII-10 — Slide-in animation polish
Spec line 599 mandates `prefers-reduced-motion` respect. Implementation:
- **(a) `transition-transform duration-150 ease-out` on the panel; matchMedia check disables it for reduced-motion users.** Recommended.
- (b) Pure transform, no transition (instant). Brutalist-purist; deviates from spec wording ("slide-in").
- (c) Longer 250ms with easing.
- **Default:** (a). Spec says slide; 150ms is the Frontfiles-tight default.

### IPIII-11 — `NAVIGATE_SIDE_PANEL` filter scope contract
The C2.1 placeholder uses `state.assetOrder` as the visible scope (reducer line 280). C2.3 must replace this with the filtered+sorted+searched view. Question: when navigating from an asset that the current filter has hidden (the user changed the filter while the panel was open), what's the cursor's starting point?
- **(a) If current open id is NOT in the filtered visible list, treat as `currentIdx = -1` and start from index 0 (first visible).** Recommended.
- (b) Bail (return state, no navigation).
- (c) Auto-close the panel on filter-induced hide.
- **Default:** (a).

### IPIII-12 — Thumbnail source
Spec §7.1 shows a ~400px square thumbnail. Source for the thumbnail image:
- **(a) `URL.createObjectURL(asset.file)` if `asset.file` is non-null** — works in dev (real File) but the dev fixtures pass `file: null` (synthetic), so the panel renders an empty placeholder square. Acceptable for visual QA.
- (b) Fall back to a styled placeholder square (gray + filename overlay) when `file` is null. Recommended on top of (a).
- **Default:** (a) + (b) — render the URL when available, placeholder otherwise.

## 5. Acceptance criteria (recap from C2-PLAN §8.2 + this directive)

- Click any `AssetRowCompact` (in Compact / Batch / Archive modes) opens the panel showing that asset's detail; row stays selected
- Click any asset row inside an expanded `StoryGroupAccordion` body opens the panel (same behavior)
- Linear mode rows do NOT open the panel (asset list left-clicks are inert for panel purposes)
- `Esc` dispatches `CLOSE_SIDE_PANEL`
- `✕` button dispatches `CLOSE_SIDE_PANEL`
- `J` dispatches `NAVIGATE_SIDE_PANEL { direction: 'next' }`; `K` dispatches `{ direction: 'prev' }`
- Navigation respects the current filter (selector is `getFilteredSortedSearchedAssets`)
- Title / Caption / Tags / Price / Privacy / Licences / Geo edits dispatch `UPDATE_ASSET_FIELD` with correct field key + value; row in the asset list reflects the change immediately (no save button anywhere)
- Exceptions section lists `getAssetExceptions(state)[id]` with inline resolver for `duplicate` (`DuplicateResolver` dispatching `RESOLVE_DUPLICATE`) and inline conflict UI for `metadata_conflict` (dispatching `RESOLVE_CONFLICT`)
- `DuplicateResolver` renders both thumbnails side-by-side with "Keep both" / "Mark this one as duplicate of that one" buttons per spec §7.2
- AI Proposal Detail section renders as collapsible shell with `// TODO: C2.5` body
- Panel pushes asset list to the left; does NOT overlay
- Slide-in 150ms transform; respects `prefers-reduced-motion`
- Stale assetId auto-closes panel
- Density transition into Linear auto-closes panel (per IPIII-3)
- All targeted tests pass; parity unchanged
