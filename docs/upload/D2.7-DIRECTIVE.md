## 1. Scope

### 1.1 In scope

- **Whole-window file-drop listener** at `UploadShellV4` root level. Listens for `dragover` + `drop` events on `window` (or the root container), filters to `dataTransfer.types.includes('Files')` so internal `@dnd-kit` drags don't fire it, dispatches `ADD_FILES` with the dropped File objects.
- **Visual drag-over feedback**: full-window dashed-border overlay with "Drop to add files" text while a file drag is over the window. State: local `isFileDragging` boolean in UploadShellV4.
- **`+ Add files` button wiring**: clicking opens a hidden `<input type="file" multiple>` programmatically. On change, dispatches `ADD_FILES`. Same handler shared with EmptyState's "or click to browse" affordance.
- **`EmptyState` augmentation**: the "or click to browse" text becomes clickable; opens the same hidden file picker.
- **New `SessionDefaultsPopover.tsx`**: anchored to the cog button. Form fields: privacy (PUBLIC/RESTRICTED/PRIVATE), licences (multi-checkbox per LicenceType), tags (comma-separated input), watermark mode (per WatermarkMode enum). Each field change dispatches `SET_DEFAULTS` with the partial update. Dismisses on click-outside, Esc, or clicking the cog again.
- **File-to-asset mapping helper** `filesToAssetDescriptors(fileList)`: maps `File[]` ‚Üí the `ADD_FILES.files` array shape. Detects format from MIME type (image ‚Üí photo; video ‚Üí video; audio ‚Üí audio; etc.). Generates `thumbnailRef` via `URL.createObjectURL` for preview-eligible files.
- Targeted vitest regression gate (173 tests still pass)

### 1.2 Out of scope (deferred)

- **Real upload pipeline** (HTTP POST, server-side processing, persistence) ‚Üí PR 5
- **URL paste / clipboard image paste** ‚Üí defer
- **Folder import preserving hierarchy** ‚Üí defer
- **Cloud-storage import** (Drive / Dropbox / Box) ‚Üí defer
- **Drag from another Frontfiles tab** ‚Üí defer
- **Watermark preview in the popover** ‚Üí defer (just the mode selector)
- **AI proposal generation on file drop** ‚Üí owned by E2 (real AI pipeline). For D2.7, dropped files have `proposal: null` until a real backend assigns one.
- **Component-test infrastructure** ‚Üí still deferred

## 4. Implementation Points (ratify before implementation)

### IPD7-1 ‚Äî Drop listener attachment point
- **(a)** Attach to a root `<div>` inside UploadShellV4 (not `window`). Cleaner unmount/cleanup when the shell unmounts. Recommended.
- (b) Attach directly to `window`.
- **Default: (a).**

### IPD7-2 ‚Äî File picker mechanism
- **(a)** A hidden `<input type="file" multiple ref={...}>` rendered inside UploadShellV4. The "+ Add files" and "click to browse" handlers programmatically call `inputRef.current.click()`. On `onChange`, read `event.target.files` and dispatch `ADD_FILES`. Recommended.
- (b) Use the modern File System Access API (`window.showOpenFilePicker`). Newer; not supported in all browsers.
- **Default: (a).**

### IPD7-3 ‚Äî File picker accept attribute
- **(a)** `accept="image/*,video/*,audio/*,.txt,.svg,.ai"` ‚Äî broad enough to cover the AssetFormat enum (photo / video / audio / text / illustration / infographic / vector). Recommended.
- (b) `accept="*/*"` ‚Äî accept anything; format detection sorts it out.
- (c) No `accept` attribute.
- **Default: (a).** Slight discoverability nudge for users picking from system dialogs.

### IPD7-4 ‚Äî Format detection from MIME
- **(a)** MIME prefix mapping: `image/* ‚Üí 'photo'`; `video/* ‚Üí 'video'`; `audio/* ‚Üí 'audio'`; `text/* ‚Üí 'text'`; `image/svg+xml ‚Üí 'vector'`; `application/postscript ‚Üí 'illustration'`. Default null otherwise. Recommended.
- (b) Always set `format: null` and let a backend pipeline classify.
- **Default: (a).** Useful for immediate visual classification (status dots, icons) without waiting for backend.

### IPD7-5 ‚Äî Thumbnail object-URL generation
- **(a)** For image files only, `URL.createObjectURL(file)` ‚Üí assigned to `thumbnailRef`. Other formats get `thumbnailRef: null` (rely on placeholder). Recommended.
- (b) Generate for all file types.
- **Default: (a).** Browsers can't render thumbs for arbitrary file types; only images preview cleanly.

### IPD7-6 ‚Äî Drop overlay visual
- **(a)** Full-window fixed overlay with `border-4 border-dashed border-blue-600` + centered "Drop to add files" text + semi-transparent dim. Inset by 16px from window edges. Recommended.
- (b) Subtle border highlight only.
- **Default: (a).**

### IPD7-7 ‚Äî Drag-over throttling
The `dragover` event fires repeatedly. Need to debounce setting `isFileDragging` true.
- **(a)** Set true on first `dragenter`, set false on `dragleave` only when `e.relatedTarget === null` (left the window) OR after a short timeout. Standard pattern. Recommended.
- (b) Use a ref counter: increment on dragenter, decrement on dragleave, set true when count > 0.
- **Default: (b).** Counter is more reliable across child-element transitions.

### IPD7-8 ‚Äî SessionDefaultsPopover anchoring
- **(a)** Absolute-positioned beneath the cog button (top: cog.bottom + 4px; left: cog.left). Bounded to stay inside the rail. Recommended.
- (b) Floating UI library (e.g., Floating UI / Popper.js).
- (c) Modal in screen center.
- **Default: (a).** Brutalist convention prefers minimal libraries.

### IPD7-9 ‚Äî SessionDefaultsPopover form fields
Per UX-SPEC-V4 ¬ß13: privacy / licences / tags / watermark.
- **(a)** All four. Form fields:
  - Privacy: `<select>` over PUBLIC/RESTRICTED/PRIVATE
  - Licences: checkbox grid (same as right rail Licences collapsible)
  - Tags: comma-separated input
  - Watermark: `<select>` over WatermarkMode enum
  - Each change dispatches `SET_DEFAULTS` with the partial update
- (b) Subset (skip watermark ‚Äî defer to its own surface).
- **Default: (a).** All four because all four are in the V2Defaults shape.

### IPD7-10 ‚Äî Popover dismiss mechanics
- **(a)** Click outside (window-level mousedown listener that checks if target is inside the popover); Esc key; clicking the cog again. Recommended.
- (b) Esc + cog only.
- **Default: (a).**

### IPD7-11 ‚Äî `isFileDragging` state location
- **(a)** Local state in UploadShellV4. The drop overlay reads it via prop or via a context. For D2.7 simplicity: local state + the overlay div is rendered inside UploadShellV4. Recommended.
- (b) Add to V3UIState. Premature; transient state doesn't belong in the reducer.
- **Default: (a).**

### IPD7-12 ‚Äî Test depth
- **(a)** No new reducer / selector tests. ADD_FILES and SET_DEFAULTS handlers are already tested. Visual smoke is the gate. Recommended.
- (b) Add a tiny test for `filesToAssetDescriptors` (pure helper; testable without DOM).
- **Default: (b).** The helper is pure logic; ~5 cases are cheap insurance.

### IPD7-13 ‚Äî Popover state location
The cog button lives in LeftRailHeader. The popover opens relative to the cog. Where does `popoverOpen` state live?
- **(a)** Local state in LeftRailHeader. The popover renders inside LeftRailHeader (anchored absolute relative to the cog). Recommended ‚Äî keeps state close to its trigger.
- (b) UploadShellV4 manages it; LeftRailHeader receives an `onCogClick` prop.
- **Default: (a).**

## 5. Acceptance criteria

- Drag a file from the desktop anywhere into the window ‚Üí drop overlay appears (dashed blue border, "Drop to add files")
- Release the drop ‚Üí asset added; appears in contact sheet (or, if Empty state, transitions to Workspace state)
- Click "+ Add files" in LeftRailHeader ‚Üí native file picker opens
- Click "or click to browse" in EmptyState ‚Üí same native file picker opens (when in Empty state)
- Pick files in the native picker ‚Üí ADD_FILES dispatched; assets appear
- Image files get `thumbnailRef` via object-URL (preview renders in contact sheet card immediately)
- Non-image files get `thumbnailRef: null` (placeholder shown ‚Äî format watermark + filename)
- Format detection per MIME: jpg ‚Üí 'photo'; mp4 ‚Üí 'video'; mp3 ‚Üí 'audio'; svg ‚Üí 'vector'; etc.
- Click cog in LeftRailHeader ‚Üí popover opens beneath the cog
- Edit privacy / licences / tags / watermark in popover ‚Üí each change dispatches `SET_DEFAULTS` with the partial update
- Click outside popover OR press Esc OR click cog again ‚Üí popover dismisses
- Internal @dnd-kit drags (card drag onto story header) do NOT trigger the file-drop overlay (gated by dataTransfer.types check)
- All 173 existing tests still pass (regression gate)
- New `filesToAssetDescriptors` helper has ~5 unit tests
